## Screenshot Guidelines

### Recommended Dimensions
- Desktop views: 1920x1080 (or 1280x720 for smaller files)
- Tablet views: 1024x768
- Mobile views: 375x667 (iPhone SE) or 390x844 (iPhone 12/13/14)

### Format
- PNG format preferred (for UI screenshots)
- JPEG acceptable for larger images
- Optimize file sizes (keep under 500KB per image)

### Content Guidelines
- Capture full page or relevant section
- Use realistic sample data (avoid Lorem Ipsum)
- Show interactive elements in various states
- Include progress indicators and feedback
- Demonstrate key features clearly

### Privacy
- Do not include real user data
- Use placeholder names and emails
- Blur or remove sensitive information

## Taking Screenshots

### Browser Tools
- **Chrome**: Press `Ctrl+Shift+P` (Win) or `Cmd+Shift+P` (Mac), type "screenshot"
- **Firefox**: Right-click → "Take Screenshot"
- **Edge**: Similar to Chrome

### Recommended Tools
- **Windows**: Snipping Tool, Greenshot
- **macOS**: Cmd+Shift+4 (partial), Cmd+Shift+3 (full)
- **Linux**: GNOME Screenshot, Spectacle

### For Mobile Screenshots
- Use Chrome DevTools Device Mode (F12 → Toggle device toolbar)
- Capture at standard mobile resolutions
- Test on actual devices when possible
